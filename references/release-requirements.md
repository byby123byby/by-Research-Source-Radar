# Release Requirements

These stable requirements define the public and safety-critical behavior covered by `RELEASE_COMPLETENESS.json`. They record the current project requirements; they are not proof that an external conversation or source was reproduced verbatim.

## REQ-DISCOVERY-001

The Skill must run broad, reproducible discovery in direct-use and mechanism-transfer lanes, record exact query executions, keep unresolved gaps visible, and avoid exhaustive-discovery claims.

## REQ-IDENTITY-002

Selected papers and repositories must resolve through authoritative metadata and pin an appropriate publication or Git object snapshot before supporting an implemented claim.

## REQ-SEED-003

A user-shared name, link, screenshot, caption, transcript, or spoken description must be represented with privacy-minimized provenance and a mechanism fingerprint before seed-to-neighbor expansion.

## REQ-TREND-004

Popular or emerging work must be discovered through dated, independently grouped attention signals that remain separate from evidence quality.

## REQ-TRANSLATION-005

Every adopted or adapted mechanism must connect its source to a decision, artifact, positive test, failure test, audit evidence, reported claim, and explicit boundary.

## REQ-SECURITY-006

Contracts, evidence, metadata requests, rendering, and writes must reject unsafe paths, malformed or oversized input, identity mismatches, unsafe redirects, secret leakage, and concurrent replacement.

## REQ-INSTALL-007

The Skill must install transactionally across documented host layouts, preserve unrelated instruction content, reject unsafe package entries, and roll back incomplete updates.

## REQ-AUDIT-008

A release may claim convergence only after the frozen artifact, tool profile, required checks, and machine-readable completeness matrix pass twice without modification; uncovered surfaces must remain explicit.

## REQ-EVAL-009

Comparative retrieval claims must use a frozen, paired, condition-isolated A/B protocol with identical prompts and budgets, condition-blind pooled source judgment, preregistered outcomes, task-level analysis, and explicit limits on generalization.
